# AIF-C01 / AWS Certified AI Practitioner 模擬試験（2 回目）

- SageMaker Model Cards

  - AI モデルの使用目的や性能、リスクなどを第三者に説明し、透明性を確保するために役立ちます。
  - 外部のステークホルダーやクライアントに向けてのドキュメントと考えていただいてもよいですが、システムの設計書のようなものなので、内部メンバーにとっても有用です。

- Amazon SageMaker Clarify の「説明可能性（Explainability）」機能: AI/ML モデルがどのように予測結果を生成しているのか、そのプロセスや根拠を明らかにするためのツールです。これにより、モデルがなぜ特定の結果を出力したのかを理解しやすくなり、特に重要なビジネスや法的判断に関わる場合に役立ちます。
- 平均二乗誤差（MSE）: まず予測値と実際の値の差を求め、それを二乗し、二乗した差の平均を計算することで得られます。これにより、誤差の大きさを正確に測定することができます。

- SageMaker Canvas: 分類、回帰、時系列予測などの機械学習モデルをサポートしています。その他ランダムカットホレスト(データセット内の異常なデータポイントを検出するための教師なしアルゴリズム)も使えました。

- ROC 曲線下面積 (AUC) : 分類精度を示す指標です。AUC（Area Under the Curve）は、ROC 曲線の下の面積を表し、モデルの予測能力を評価する指標です。特に、モデルが陽性クラスと陰性クラスをどれだけうまく区別できるかを測ります。AUC の値は 0.5 から 1.0 の範囲を取り、1 に近いほどモデルの分類性能が良いことを示します。

- SageMaker Feature Store の特徴量グループとは、関連する特徴量（データの属性や特徴）をまとめた論理的な単位です。特徴量グループは、データセットの特定の部分を表し、例えば「ユーザーのプロフィール情報」や「商品の属性」といった、モデルで使うデータの集合を整理して管理するために使用されます。

- Amazon Bedrock のモデル評価には、自動モデル評価ジョブとヒューマンワーカーによるモデル評価ジョブの 2 種類があります。自動モデル評価ジョブでは、組み込みのプロンプトデータセットと独自のカスタムプロンプトデータセットの両方を使用できます。一方、ヒューマンワーカーによるモデル評価ジョブでは、独自のデータセットを使用する必要があります。
